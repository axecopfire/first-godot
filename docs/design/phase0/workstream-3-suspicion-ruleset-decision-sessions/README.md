## Purpose

Use this series to convert Workstream 3 mechanics intent into unambiguous design rules.

Run sessions in sequence. Each session resolves one rules layer and exports decisions into final Workstream 3 artifacts.

## Required outputs this series must produce

* Suspicion Event Catalog v1
* Social Norm Rulebook v1
* Threshold Reaction Playbook (35, 60, 80)
* Recovery and Rumor Rules Sheet

## Session sequence

1. Session 1: Suspicion event dictionary and severity bands
2. Session 2: Social mismatch heuristics and norm inference rules
3. Session 3: Threshold reactions and witness escalation behavior
4. Session 4: Recovery rules and rumor propagation classes
5. Session 5: Cross-artifact consistency review and sign-off

## How to run each session

* Include narrative, systems, and world design in the room
* Record one explicit decision per prompt
* Convert disagreements into open issues with owner and due date
* Mark unresolved high-impact items as Milestone A blockers
* End by exporting accepted decisions into the target artifact section

## Guardrail checks

1. Does this rule model suspicion as social assimilation pressure first, then enforcement pressure?
2. Does this rule avoid revealing the hidden identity premise too early?
3. Does this rule preserve at least one non-stealth way to recover suspicion through social learning or sponsorship?
4. Does this rule reinforce language learning and trust-building as core progression?

## Decision quality bar

A decision is complete only if all checks pass:

* Trigger conditions are concrete and testable
* Severity is explicit and uses consistent bands
* Witness scope is defined: local, area, institution, or global
* Recovery path is defined now or explicitly deferred
* Downstream dependency impact is noted for Workstreams 1, 2, and 5

## Exit criteria for Workstream 3

* Event dictionary covers the complete Phase 1 action set
* Social mismatch rules are consistent across locations and NPC roles
* Threshold reactions at 35, 60, and 80 are unambiguous and actor-specific
* Recovery and rumor rules are complete enough for Workstream 1 and 2 dependency usage
* Remaining open issues are non-blocking and time-boxed
