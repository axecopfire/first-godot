## Purpose

Use this questionnaire to convert Workstream 1 goals into concrete design decisions.

Complete it before finalizing the Location Design Matrix and Spatial Contract Map.

Workstream 1 must review both existing locations and net-new proposals. Existing locations are not exempt.

## How to use

* Run this as a facilitated session with narrative, systems, and world design represented
* Record one clear answer per question, even if temporary
* Mark unknowns as open items with an owner and due date
* Treat unresolved high-risk questions as blockers for Milestone A
* Run a full backfill pass across every currently implemented location before net-new prioritization

## Premise and guardrail checks

1. Does this location reinforce early-game social assimilation pressure, not only stealth pressure?
2. What human-normal behavior is expected in this space, and how will players learn it?
3. What non-stealth path in this space can lower suspicion through social learning or sponsorship?
4. What design detail here must avoid revealing the hidden player-identity premise too early?

## Section 0: Backfill and normalization gate

Complete this gate before scoring net-new proposals.

1. Have all currently implemented locations been profiled with Sections A through D?
2. Which existing locations are `Keep`, `Keep with revision`, `Candidate cut`, or `Deferred`?
3. Which existing locations duplicate role coverage and should be merged or removed?
4. Which existing locations fail to support either a pressure path or recovery path?
5. Which existing locations require revision before Milestone A can be considered ready?

## Section B: Witness and visibility model

1. Who are the expected witnesses in this location by role and institution?
2. Which witness types are always present, and which are conditional by time or event?
3. Where are the dominant line-of-sight lanes, and where is meaningful cover?
4. Which actions are socially appropriate, suspicious, or severe in this location?
5. Which suspicion thresholds (35, 60, 80) should become visible here first?
6. What immediate NPC reaction should occur at each threshold in this space?

## Section C: Entry, access, and social appropriateness bands

1. What are the allowed, discouraged, and forbidden entry conditions?
2. What changes based on time-of-day, weather, or institution state?
3. Which NPC, token, or sponsorship is required for low-friction entry?
4. What language level or social knowledge is required to avoid mismatch penalties?
5. Which player actions are context-inappropriate here, and what deltas should they trigger?

## Section D: Adjacency and traversal contracts

1. Which locations connect directly to this one, and what is the intended route hierarchy?
2. Does this location create a dead end for any core loop branch?
3. Which transition should feel safe, risky, or ambiguous, and why?
4. Where should chokepoints exist for institution control and for stealth alternatives?
5. Which adjacency supports recovery after a failure or arrest state?
6. What route options preserve pacing between social, stealth, and survival contexts?

## Section F: Artifact handoff readiness

Complete this checklist before marking Workstream 1 foundation output as done.

* Every location has a primary and secondary gameplay role
* Every high-risk location has defined witness density and threshold reaction intent
* Every critical loop beat maps to at least one low-risk and one high-risk location
* Every location has at least one recovery path and one pressure path
* Adjacency contracts are explicit enough for implementation task slicing
* Open questions have owners, due dates, and fallback assumptions

## Output package expectations

At session end, export answers into the following artifacts:

* High-level Location Design Matrix
* Per-location profile pack (one file per location)
* Spatial Contract Map
* Location Readability Guide

If one of these cannot be produced from the answers, rerun the missing section before closing Milestone A.
